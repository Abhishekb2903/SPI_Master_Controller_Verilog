# SPI Master Controller using Verilog HDL (RTL Design)

## Overview

Designed and implemented an SPI (Serial Peripheral Interface) Master Controller using Verilog HDL and verified functionality through behavioral simulation in Xilinx Vivado. The controller performs synchronous serial data transmission of 8-bit frames using MOSI, SCLK, and Slave Select (SS) control signals.

This project demonstrates FSM-based protocol implementation and clock-driven serial communication architecture used in FPGA and ASIC designs.

---

## Features

- 8-bit serial data transmission (MSB first)
- Master-controlled clock generation (SCLK)
- Slave Select (SS) signal control
- MOSI serial data shifting
- Transmission completion indicator (done signal)
- Finite State Machine (FSM) based control logic
- Behavioral simulation verification in Vivado

---

## SPI Protocol Signals Implemented

| Signal | Direction | Description |
|--------|-----------|-------------|
| SCLK | Master → Slave | Serial clock generated by master |
| MOSI | Master → Slave | Serial data output |
| SS | Master → Slave | Slave enable control |
| done | Output | Transfer completion indicator |

---

## Design Architecture

The SPI Master Controller operates as a synchronous FSM performing the following sequence:


Start Signal Asserted
↓
Slave Select Activated (SS LOW)
↓
Serial Clock Generated (SCLK toggling)
↓
8-bit Data Shifted via MOSI (MSB first)
↓
Slave Select Deactivated (SS HIGH)
↓
Transfer Completion Flag Asserted


---

## FSM Operation

The controller transitions through sequential states to:

- Capture input data
- Enable slave device
- Shift each data bit serially
- Toggle serial clock
- Indicate transmission completion

This ensures deterministic synchronous communication between master and slave devices.
